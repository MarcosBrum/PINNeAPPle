Why Physics-Informed Neural Networks?

Bridging the gap between data-driven ML and first-principles physics
⚛

Physics Constraints

Encode governing PDEs (Navier-Stokes, Maxwell, Elasticity) directly in the loss function. The network is forced to respect physical laws, enabling extrapolation beyond training data.

📊

Data Efficiency

Physical priors dramatically reduce data requirements. PINNs can solve forward/inverse problems with sparse or noisy observations where classical methods fail.

∂

Differentiability

Automatic differentiation provides exact derivatives at any point. Enables gradient-based design optimization, sensitivity analysis, and uncertainty propagation.

Classical Solvers (FEM / FDM)

  • Require mesh generation (expensive for complex 3D)
  • Fixed resolution — refining is costly
  • Inverse problems require adjoint derivation
  • Not differentiable w.r.t. geometry or BCs
  • Parallelism limited by mesh decomposition

PINNs (PINNeAPPle)

  • Mesh-free — geometry via SDF or CSG
  • Adaptive collocation, resolution-free inference
  • Inverse problems solved jointly with forward
  • Fully differentiable w.r.t. all inputs/params
  • GPU/multi-GPU / JAX / distributed training

Engineering Use Cases

Aerodynamics (NACA, turbomachinery) Heat Transfer Structural Mechanics Electromagnetics Fluid-Structure Interaction Turbulence Modeling Inverse Material ID Shape Optimization

pinneaple_data — Collocation & Active Learning

from pinneaple.data import (
    CollocationSampler, ResidualBasedAL
)

# Latin-hypercube interior + boundary samples
sampler = CollocationSampler(
    domain={"x": (0,1), "t": (0,1)},
    n_interior=10_000,
    n_boundary=2_000,
    strategy="lhs"   # or "uniform", "sobol"
)
x_int, x_bc = sampler.sample()

# Active learning: refine around high residuals
al = ResidualBasedAL(
    model=model, pde_fn=spec.pde_fn,
    n_candidates=50_000, top_k=2_000
)
x_new = al.refine(x_int)   # returns high-error pts

Sampling Strategies

  • Uniform random — baseline Monte Carlo
  • Latin Hypercube (LHS) — space-filling, low discrepancy
  • Sobol sequences — quasi-random, best coverage
  • Residual-based AL — concentrate near PDE violation
  • Gradient-based AL — concentrate near high-gradient regions

pinneaple_pinn — Physics Loss Compiler

from pinneaple.pinn import PINNFactory, DoMINO

# Build PINN loss from spec
factory = PINNFactory(spec=spec, model=model)
loss_fn = factory.compile(
    weights={"pde": 1.0, "bc": 10.0, "ic": 5.0}
)

# DoMINO domain decomposition (v0.5)
domino = DoMINO(
    domain=flow_domain,
    n_subdomains=4,
    overlap=0.1
)
subdomains = domino.partition()

# Each subdomain gets its own PINN
pinns = [PINNFactory(spec, model=MLP(2,1))
         for _ in subdomains]

# Interface continuity conditions auto-enforced
loss = domino.coupled_loss(pinns)

DoMINO Domain Decomposition

How It Works

  • Partitions domain into N overlapping subdomains
  • Each subdomain trains a separate PINN network
  • Interface conditions: u₁(x) = u₂(x) at boundaries
  • Flux continuity: ∇u₁·n = ∇u₂·n enforced as loss term
  • Enables solving large domains that don't fit in one PINN

PINNFactory Features

  • Symbolic compilation of PDE residuals
  • Auto weight balancing (NTK, GradNorm, ReLoBRaLo)
  • HardBC & PeriodicBC enforcement
  • Adaptive loss weighting during training

Loss Terms

  • L_pde — PDE residual at interior pts
  • L_bc — boundary condition mismatch
  • L_ic — initial condition (time-dep PDEs)
  • L_data — optional data fitting term

Feature 1 — Symbolic PDE Compiler

Write PDEs as mathematics. PINNeAPPle compiles them to autograd automatically.

Before (v0.4 — hardcoded)

# Manual if-elif chains for each PDE
def poisson_residual(model, x):
    u = model(x)
    u_x = torch.autograd.grad(
        u, x, ..., create_graph=True)[0][:,0]
    u_xx = torch.autograd.grad(
        u_x, x, ...)[0][:,0]
    u_y = torch.autograd.grad(
        u, x, ...)[0][:,1]
    u_yy = torch.autograd.grad(
        u_y, x, ...)[0][:,1]
    f = 2*pi**2*sin(pi*x[:,0])*sin(pi*x[:,1])
    return u_xx + u_yy + f

After (v0.5 — symbolic)

import sympy as sp
import pinneaple as pp

x, y = sp.symbols('x y')
u = sp.Function('u')

# Poisson: ∇²u + 2π²sin(πx)sin(πy) = 0
pde = pp.SymbolicPDE(
    u(x,y).diff(x,2) + u(x,y).diff(y,2)
    + 2*sp.pi**2*sp.sin(sp.pi*x)*sp.sin(sp.pi*y),
    coord_syms=[x, y], field_syms=[u(x,y)]
)
# pde.to_autograd() → callable residual function
residual_fn = pde.to_autograd()

Features 4 & 5 — Hard BC & Periodic BC Enforcement

Hard Boundary Conditions

Ansatz: φ(x)·net(x) + g_bc(x)

The distance function φ(x) vanishes at boundaries, so the prediction automatically satisfies the BC — no BC loss term needed. Eliminates BC weight tuning.

from pinneaple.pinn import HardBC

# φ(x) = x(1-x) for [0,1] Dirichlet
hard = HardBC(
    phi=lambda x: x*(1-x),
    g_bc=lambda x: torch.zeros_like(x[:,0])
)
model_with_bc = hard.wrap(base_model)
# Now model_with_bc(x) always satisfies u=0 at x=0,1

Periodic Boundary Conditions

Transform: x → (cos(2πx/L), sin(2πx/L))

Coordinate transformation ensures network output is automatically periodic with period L. No periodicity loss term required. Works for multi-dimensional periodic domains.

from pinneaple.pinn import PeriodicBC

# Periodic in x with period L=2π
pbc = PeriodicBC(
    periodic_dims=[0],    # x-axis
    periods=[2*np.pi]
)
model_periodic = pbc.wrap(base_model)
# model_periodic(x) == model_periodic(x + 2π)

Benefits

  • Hard BCs: eliminates weight tuning for BCs
  • Periodic BCs: exact periodicity, not approximate
  • Faster convergence — network focuses on interior physics
  • Better conditioning of the optimization problem

Feature 2 — SIREN: Sinusoidal Representation Networks

from pinneaple.models import SIREN

model = SIREN(
    in_features=2,      # (x, t)
    out_features=1,     # u(x,t)
    hidden_layers=4,
    hidden_dim=256,
    omega_0=30.0        # frequency scale
)
# SIREN activation: sin(ω₀ · Wx + b)
# Derivatives are also sines → smooth everywhere

# Special initialization (Sitzmann et al. 2020)
# Layer 0: U[-1/n, 1/n]
# Hidden:  U[-√6/(n·ω₀), √6/(ω₀√n)]

Weight Initialization

  • Layer 0: U[−1/n, 1/n] — preserves input distribution
  • Hidden: U[−√6/(n·ω₀), √6/(ω₀·√n)] — preserves arcsine distribution
  • Ensures output distribution is uniform at initialization
  • Critical: wrong init → training failure

Why SIREN for PDEs?

  • Derivatives of sin are also sin — preserves smoothness through all orders
  • Standard tanh/ReLU networks struggle with high-frequency content
  • SIREN represents signals at any frequency (controlled by ω₀)
  • Exact derivatives at all orders via autograd

Best Use Cases

  • Wave equations (Helmholtz, acoustic)
  • High-frequency oscillatory solutions
  • Scattering problems
  • Seismic wave propagation
  • Electromagnetic fields

Feature 3 — Modified MLP (Wang et al. 2022)

from pinneaple.models import ModifiedMLP

model = ModifiedMLP(
    in_features=2,
    out_features=1,
    hidden_dim=128,
    n_layers=6,
    fourier_scale=1.0,   # Fourier embedding σ
    n_fourier=64          # Fourier features dim
)

# Architecture:
# 1. Random Fourier Features: γ(x) = [cos(Bx), sin(Bx)]
# 2. U = σ(W_1 · γ(x) + b_1)  [highway branch U]
# 3. V = σ(W_2 · γ(x) + b_2)  [highway branch V]
# 4. Each hidden: H = σ(WH+b)*(1-V) + U*V
# 5. Output = W_out · H_L

Key Design — Highway U/V Branches

The U and V branches act as learned gating signals. At each hidden layer: H_{l+1} = σ(W_l H_l + b_l) ⊙ (1−V) + U ⊙ V. This creates information highways that help gradient flow through deep networks while the Fourier embedding handles multi-scale behavior.

When to Use ModifiedMLP

  • Multi-scale solutions with sharp gradients
  • High-dimensional PDEs (d > 4)
  • Long-time integration where spectral bias is severe
  • Turbulent flow fields with multiple active scales

Window IC Transfer

After window k finishes, the model is evaluated on a dense grid at t = k·ΔT. These values become the initial condition for window k+1. The network is re-initialized or fine-tuned for each window.

  • 3–10× lower error vs full-time PINN on same epochs

Feature 14 — Causal PINN Training (Wang et al. 2022)

from pinneaple.train import CausalPINNTrainer

trainer = CausalPINNTrainer(
    model=model,
    spec=wave_spec,
    epsilon=1.0,     # causality tolerance
    n_time_bins=100, # temporal bins
    max_epochs=100_000
)

# Causal weights per time bin k:
# w_k = exp(-ε · Σ_{j=0}^{k-1} L_j)
# If early bins have high loss → late bins get
# near-zero weight → network focuses on early time

trainer.fit()

# Monitor causal weight evolution
weights = trainer.get_causal_weights()
# Should progress from [1,0,0,...] to [1,1,...,1]

The Causality Problem in PINNs

Standard PINNs treat all time bins equally. This causes "future cheating" — the network learns a low-loss equilibrium that satisfies the PDE everywhere but violates causality. The solution at t=1.0 can influence training at t=0.1.

  • w_k = exp(−ε·Σ_{j<k} L_j) → early-time losses must be small before late-time bins are activated
  • ε controls strictness: large ε = strict causality, small ε = relaxed
  • Weights evolve during training: initially [1,~0,~0,…] → [1,1,1,…]

When Causal Training Helps

  • Convection-dominated PDEs (Burgers at high Pe)
  • Wave equations with complex interference patterns
  • Reaction-diffusion with sharp transients
  • Any PDE where solution at t+dt depends critically on t

from pinneaple.design_opt import (
    ContinuousAdjointSolver,
    ShapeParametrization,
    DragAdjointObjective
)

# NACA airfoil aerodynamic optimization
shape = ShapeParametrization.naca(
    series="0012",
    n_control_pts=20   # FFD control points
)

    pinn=ns_pinn, alpha=5.0  # angle of attack
)

adj = ContinuousAdjointSolver(
    pinn=ns_pinn,
    objective=obj,
    shape=shape
)

# dJ/d(shape) — exact gradient via adjoint
sensitivity = adj.sensitivity()

# Gradient descent step
for i in range(50):
    dJ = adj.sensitivity()
    shape.update(step=-0.005*dJ)
    ns_pinn.retrain(shape=shape, epochs=2000)

Continuous Adjoint Method

  • Objective J = ∫ f(u, shape) dΩ (e.g. drag force)
  • Adjoint PDE: L*ψ = −∂f/∂u (backward solve)
  • Sensitivity: dJ/ds = ∫ ψ · ∂R/∂s dΩ + boundary terms
  • Exact gradient — no finite differences needed
  • Cost: 1 forward solve + 1 adjoint solve (O(1) in shape dim)

ShapeParametrization (FFD)

  • Free-Form Deformation control lattice around geometry
  • n_control_pts: 10–50 for practical optimization
  • Shape space much smaller than full mesh node coords
  • Smooth deformation preserving geometry regularity

Applications

  • Airfoil drag minimization (lift-constrained)
  • Heat exchanger fin shape optimization
  • Nozzle geometry for maximum thrust

Getting Started with v0.5

5-Minute Quickstart

# Install
pip install pinneaple

import pinneaple as pp
import sympy as sp

# 1. Define PDE symbolically
x, t = sp.symbols('x t')
u = sp.Function('u')
pde = pp.SymbolicPDE(
    u(x,t).diff(t) + u(x,t)*u(x,t).diff(x),
    coord_syms=[x, t], field_syms=[u(x,t)]
)

# 2. Build model
model = pp.get_model("SIREN",
    in_features=2, out_features=1)

# 3. Sample collocation points
sampler = pp.CollocationSampler(
    domain={"x": (-1,1), "t": (0,1)},
    n_interior=10_000, n_boundary=2_000
)

# 4. Compile PINN loss
factory = pp.PINNFactory(pde, model)
loss_fn = factory.compile()

# 5. Train
trainer = pp.Trainer(model, loss_fn)
trainer.fit(epochs=20_000)
