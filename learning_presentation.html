Your First PINN — Harmonic Oscillator

x″(t) + ω²x(t) = 0  ·  x(0)=1, x′(0)=0  ·  Exact: cos(ωt)

Key PINNeAPPle APIs

from pinneaple_symbolic import SymbolicPDE, HardBC
import sympy as sp

# Step 1 — Define PDE symbolically
t, x = sp.Symbol('t'), sp.Function('x')
pde = SymbolicPDE(
    expr       = x(t).diff(t,2) + omega**2 * x(t),
    coord_syms = [t], field_syms = [x],
)
residual_fn = pde.to_residual_fn(net)

# Step 2 — Hard-enforce IC: x(0)=1 via ansatz
#          x_wrapped(t) = 1 + t·net(t)  → always 1 at t=0
hard_ic = HardBC(
    distance_fn = lambda t: t,
    bc_value_fn = lambda t: torch.ones_like(t),
)
net_wrapped = hard_ic.wrap_model(net)

# Step 3 — Training loss (position IC enforced for free)
r       = residual_fn(t_col)         # PDE residual
loss_vel = (xd_at_0 - 0).pow(2)     # velocity IC only
loss = r.pow(2).mean() + 200 * loss_vel

What SymbolicPDE does for you

  • Write the PDE once in SymPy — no manual autograd chains
  • to_residual_fn(net) returns a closure that evaluates F[u_θ]
  • Works for any ODE/PDE expressible in SymPy
  • HardBC removes L_pos entirely — one fewer hyperparameter

Inverse Problem — Recovering Diffusivity

u_t = α u_xx · unknown α · 50 noisy observations · two recovery methods

Approach 1 — Gradient-based (α as nn.Parameter)

# α is a learnable parameter — same as network weights
alpha = nn.Parameter(torch.tensor([0.2]))  # wrong init

opt = Adam([*net.parameters(), alpha], lr=1e-3)

loss = l_pde(alpha) + 100*l_ic + 1000*l_data
# Gradient flows through BOTH net weights AND alpha
alpha.data.clamp_(1e-4, 1.0)  # keep physical

Approach 2 — EKI (derivative-free)

from pinneaple_inverse import (
    EnsembleKalmanInversion, EKIConfig
)

# Only need: forward_fn(theta) → u predictions
eki_cfg = EKIConfig(
    n_ensemble=20, n_iterations=15,
    noise_std=0.01, init_spread=0.5,
)
eki = EnsembleKalmanInversion(
    forward_fn=forward_model, config=eki_cfg
)
history = eki.solve(y_obs=u_obs, theta_init=theta0)
alpha_eki = exp(history.theta_mean_history[-1])

Time-Marching — Van der Pol Stiff Oscillator

x″ − μ(1−x²)x′ + x = 0 · μ=3 (stiff) · T=15 · Standard PINN fails

TimeMarchingTrainer

from pinneaple_train import TimeMarchingTrainer

# Divide [0, 15] into 15 windows of 1 time unit
# Each window: fresh network + IC from previous end

trainer = TimeMarchingTrainer(
    model_factory     = make_net,       # fresh per window
    loss_factory      = make_loss,      # factory(ic, t0, t1)
    t_start           = 0.0,
    t_end             = 15.0,
    n_windows         = 15,           # 1s each
    epochs_per_window = 3_000,
    device            = "cuda",
    ic_state          = torch.tensor([2.0, 0.0]),
)
t_all, x_all = trainer.integrate()
# Returns full-domain prediction by chaining windows

Why standard PINN fails for μ=3

  • Stiff ODE: fast and slow time scales coexist
  • PDE loss landscape very ill-conditioned over [0,15]
  • Gradient vanishes at early times when training whole domain
  • Time-marching: each 1s window is well-conditioned
  • CausalPINNTrainer is an alternative (causal weighting)

Physics Validation — Never Skip This

Laplace ∇²u=0 · Well-trained vs under-trained · PhysicsValidator catches the bad one

PhysicsValidator — fluent API

from pinneaple_validate import (
    PhysicsValidator, compare_to_analytical
)

validator = (
    PhysicsValidator(
        model=net,
        coord_names=["x", "y"],
        domain_bounds={"x":(0,1), "y":(0,1)},
    )
    .add_boundary_check(kind="dirichlet", value=0,
                         boundary="x=0", x_bc=bnd_pts)
    .add_boundary_check(kind="dirichlet", value_fn=g,
                         boundary="x=1", x_bc=bnd_right)
)
report = validator.validate()

# Three key metrics
m = compare_to_analytical(
    model=net, analytical_fn=exact,
    coord_names=["x","y"],
    domain_bounds={...}, n_points=20_000,
)
# m["rel_l2"], m["rmse"], m["max_error"]

Why validate separately from training loss?

  • Training loss = value at training points only
  • Validation uses 10–20× more held-out points
  • max_error reveals local failures (corners, BCs) that L2 hides
  • BC residual check: model may satisfy PDE but violate BC
  • Always run before claiming model is ready

Operator Learning — Parametric Heat Family

Train on α ∈ {0.01…0.5} · Query any α instantly · No re-training

Parametric network: input (x, t, α)

from pinneaple_models import FourierFeatureEmbedding

class ParametricHeatNet(nn.Module):
    def __init__(self):
        # Spatial/temporal features: Fourier embed of (x,t)
        self.embed_xt  = FourierFeatureEmbedding(
            in_dim=2, n_fourier=32, sigma=3.0
        )
        # Parameter encoder: α → 32-dim latent
        self.param_enc = nn.Sequential(
            nn.Linear(1, 32), nn.Tanh(),
            nn.Linear(32, 32), nn.Tanh(),
        )
        self.trunk = nn.Sequential(
            nn.Linear(64+32, 128), nn.Tanh(),
            nn.Linear(128, 128), nn.Tanh(),
            nn.Linear(128, 1),
        )
    def forward(self, x, t, alpha):
        ft = self.embed_xt(cat([x, t], dim=-1))
        fa = self.param_enc(alpha)
        return self.trunk(cat([ft, fa], dim=-1))

Key insight: PINN = one problem · Operator = many

  • PINN: fixed α → needs re-training for new α (minutes/hours)
  • Operator: all α → forward pass for any α (milliseconds)
  • The parameter encoder learns a smooth α ↦ solution map
  • Interpolation between training α values works well
  • For larger families: use FNO (templates/19)
